# 💼 Escrow Account Ledger — Professional Ledger Suite

[![Live Production](https://img.shields.io/badge/Live-Production-emerald?style=for-the-badge&logo=firebase)](https://esrow-ledger.web.app)
[![Tech Stack](https://img.shields.io/badge/Vite_/_React_/_TypeScript-blue?style=for-the-badge&logo=react)](https://react.dev)
[![Database](https://img.shields.io/badge/Supabase_/_PostgreSQL-green?style=for-the-badge&logo=supabase)](https://supabase.com)

An elite, keyboard-first, desktop-grade double-entry accounting suite built to manage high-volume transactional ledgers, automate Monday Final settlements, calculate volumes, and keep full multi-party balance sheets completely in sync.

---

## ✨ Key Features & UX Highlights

*   🎹 **Keyboard-First Ledger Navigation**: Engineered for lightning-fast double-entry manual bookkeeping. Includes auto-completion overlays and custom `TAB` selectors.
*   ⚡ **Monday Final Settlement Engine**: Automates the transition of open transaction logs into permanent finalized logs weekly with strict read-only ledger locking.
*   📊 **Visual Balance Sheet**: Live rendering of Credit (Jama / Dena) and Debit (Name / Lena) balances with custom color-coded indicators and pagination filters.
*   🌗 **Seamless Global Theme Toggle**: High-performance theme provider toggles between a crisp Light Mode (default) and a stunning dark glassmorphism theme instantly.
*   👑 **Secure Admin Command Panel**: Global workspace metrics (realtime count of registered users, parties, volume logs) with active system permissions management.
*   🎯 **Automated Database Triggers**: Realtime database functions manage settlement constraints and automatically flag anomalies dynamically on the Supabase/PostgreSQL layers.

## 🛠️ Environment Variables Configuration (.env)

The application requires several environmental variables to run correctly. Copy **[.env.example](file:///c:/Users/thaku/OneDrive/Desktop/Work/Escrow%20Account%20Ledger/.env.example)** to create your **`.env`** file:

```bash
cp .env.example .env
```

Below is the detailed description of each required environment key:

### 1. General Backend Configuration
*   `VITE_API_BASE_URL`: The base URL pointing to the auxiliary transaction processing API server.
    *   *Default*: `https://your-backend-api-url.com/api`

### 2. Firebase Server Integration (Authentication & SDK)
Used for robust user account signup, password resets, and user sessions.
*   `VITE_FIREBASE_API_KEY`: The primary API key generated by your Firebase app project console.
*   `VITE_FIREBASE_AUTH_DOMAIN`: Auth domain used for OAuth redirect flows (e.g. `your-app.firebaseapp.com`).
*   `VITE_FIREBASE_PROJECT_ID`: The unique text identifier of your Firebase project.
*   `VITE_FIREBASE_STORAGE_BUCKET`: Storage bucket URL used for billing assets or invoices.
*   `VITE_FIREBASE_MESSAGING_SENDER_ID`: Unique ID for Firebase Cloud Messaging streams.
*   `VITE_FIREBASE_APP_ID`: Internal app registration ID (e.g. `1:XXXXXXXX:web:XXXXXXXXX`).
*   `VITE_FIREBASE_MEASUREMENT_ID`: Google Analytics measurement ID (prefixed with `G-`).

### 3. Supabase Realtime Database Configuration
Provides Postgres database queries, schema subscriptions, and transaction logging.
*   `VITE_SUPABASE_URL`: The URL endpoint of your Supabase server instance (e.g. `https://xxx.supabase.co`).
*   `VITE_SUPABASE_ANON_KEY`: The public API token used for standard client-side queries safely.
*   `VITE_SUPABASE_SERVICE_ROLE_KEY`: **Keep secret!** High-level bypass token used by admin dashboards to override user restrictions.

## 🚀 Complete Step-by-Step Installation Guide

Follow these exact steps to set up the project locally from scratch:

### Step 1: Pre-requisites
Ensure you have the following installed on your machine:
*   [Node.js (v18.0.0 or higher)](https://nodejs.org)
*   [Git CLI](https://git-scm.com)
*   A Firebase Account (free tier) & Supabase Account (free tier)

### Step 2: Clone the Project
```bash
git clone https://github.com/Aadarsh2021/Escrow-Account-Ledger.git
cd Escrow-Account-Ledger
```

### Step 3: Install Node Packages
```bash
npm install
```

### Step 4: Setup Environment Credentials
1. Copy `.env.example` file and rename it to `.env`.
2. Populate the actual credentials corresponding to your Firebase and Supabase project configurations (see details above).

### Step 5: Configure Supabase Database Schema
To prepare your database, open your **Supabase Dashboard**, go to **SQL Editor**, and run the schemas located in **[supabase/](file:///c:/Users/thaku/OneDrive/Desktop/Work/Escrow%20Account%20Ledger/supabase/)**:
1. Run `supabase_schema.sql` to initialize core profile schemas.
2. Run `supabase_parties_schema.sql` and `supabase_transactions_schema.sql` to construct ledgers and accounting relations.
3. Execute `supabase_monday_final.sql` and `supabase_lock_settlement.sql` to configure the locking mechanisms and automate triggers.

### Step 6: Start Local Development Server
Launch Vite's development compilation with active Hot Module Replacement (HMR):
```bash
npm run dev
```
Open your browser and navigate to `http://localhost:5173`.

### Step 7: Build for Production & Testing
Compile type-safe structures and bundle optimal static code:
```bash
npm run build
```
This builds an optimized production distribution in the `dist/` directory.

### Step 8: Deploy Live to Firebase Hosting
Deploy your local production build live instantly:
1. Log in to Firebase CLI (if not already logged in):
   ```bash
   npx firebase login
   ```
2. Deploy code:
   ```bash
   npx firebase deploy --only hosting
   ```
